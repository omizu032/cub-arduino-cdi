# Arduino Nano デジタルCDI 開発技術リファレンス

本ドキュメントは、ベースとなる「[ほいほい堂blog](https://www.hoihoido.com/blog/howtomakeacdiforcub/)」の設計に対する、8ビットマイコン（Arduino Nano, ATmega328P / 16MHz）を使ったデジタルCDIの**個人的的な再構築**の詳細について、技術的詳細を記録したものです。

## 1. 割り込み処理（ISR）の安定化

点火タイミング割り込みをネイティブ・アセンブラ記述（ISR_NAKED）で処理しているオリジナルのソースコードに対して、下記の修正を行い安定化を図りました。

- **Timer1オーバーフロー割り込み内の命令修正**
  パルスジェネレータからの点火パルスの検知から出力までのレイテンシの改善のため、Timer1オーバーフロー割り込み内のレジスタ命令 `lds` を `ldi` へ変更。
- **多重割り込みの排除**
  「割り込み処理中の別割り込み発生（スタックオーバーフロー）」による高回転時にパルスが踊る（ジッターが発生する）のを改善するために、全ISRの末尾の `sei` 命令を削除し、安全性を確保。

## 2. 8ビットマイコンの限界突破：固定小数点演算ロジック

浮動小数点演算（FPU）を持たないArduino Nanoで `float` や `long` の除算を行うと、数百サイクルの処理落ちが発生します。これを防ぐため、すべての演算を「乗算とビットシフト」に置き換えました。

- **直線補間の固定小数点化（角度計算）**
  従来：`(tmap[1] - tmap[0]) * (rpm - 1000) / 2000`
  最適化：除算を排除し、定数を乗じてシフトする方式へ変更。さらに /16度（0.0625度）の解像度（サブディグリー）を持たせるため `>> 16` から `>> 12` へ拡張し、整数切り捨てによる階段状の量子化ノイズ（飛び飛びのパルス）を改善。全回転域においてシームレスな進角を実現。
- **ディレイ値（タイマー幅）の固定小数点化**
  従来：`sdeg * averagetime / 360`
  最適化：182 / 65536 ≒ 1 / 360 であることを利用し、16ビット固定小数点定数「182」を採用。
  数式：`((long)sdeg * averagetime * 182) >> 16` （サブディグリー化に伴い `>> 20` へ調整）。

## 3. 高解像度 $\alpha - \beta$ フィルタ（仮想フライホイールモデル）

単気筒エンジン特有の「燃焼行程（速い）」「排気行程（遅い）」の脈動やノイズを吸収するため、定常状態推定フィルタ（ $\alpha - \beta$ フィルタ）を実装しました。

- **パラメータ設定**
  位置ゲイン $\alpha$ = 1/2、速度ゲイン $\beta$ = 1/8。
- **高解像度化（桁落ち防止）**
  ArduinoのTimer1分解能（4μs/count）では、微小な加速が `>> 4` 等のビットシフトによってゼロに切り捨てられ（不感帯）、結果として「1秒ごとのパルス飛び」が発生していました。これを防ぐため、内部変数を256倍（`<< 8`）スケールで保持・計算し、出力時に `>> 8` で戻すことで、浮動小数点を使わずに連続性を確保しました。

## 4. 進角タイマー（`widthB`）の位相遅れと「2手先予測」

CDIの進角は「現在のパルスジェネレータからのパルスを起点とし、ほぼ1回転待ってから発火する」仕組みです。そのため、「次のパルスまでの時間」ではなく、「次の次のパルスまでの時間」を計算する「2手先予測」を実装しました。

- **位相遅れの解消**
  1手先の予測では、減速時にタイマーが短くなりすぎ、境界領域で30〜40μsの「とび（ギャップ）」や「左側への張り付き（過進角）」を引き起こしていました。
- **解決策**
  フィルタの速度成分（`velocity`）を2倍して足し込む「2手先予測」を実装。
  数式：`predicted_interval = (ab_interval_fp + 2 * ab_velocity_fp) >> 8;`
  これにより、1サイクル分の遅れを改善。

## 5. ハードウェア慣性とシミュレーションの境界（トレードオフの理解）

パルスシミュレータによるベンチテストにおいて、2500rpm → 4500rpm（3秒間）などの急激なスイープでスロットルを全閉（折り返し）にした際、フィルタの「加速の慣性」が暴走して一時的に進角側へ大きく変動する現象が確認されました。

- **ソフトウェアの限界と物理的現実**
  進角側への大きな変動を防ぐための強烈なキャンセラーを実装すると、今度は $\alpha - \beta$ フィルタの学習が阻害され「進角側への張り付き」が再発するトレードオフ（デグレード）に直面しました。
  しかし、パルスシミュレータ（慣性ゼロ）とは異なり、実際のエンジンには「重いフライホイール」が存在します。物理法則上、4500rpmから一瞬でV字に折り返して減速することはあり得ず、U字の滑らかな減速曲線を描くため、実車環境においてはこの「変曲点ジャンプ」は発生しないか、極めて小さくなります。
- **結論**
  ソフトウェア上での過度な「ジャンプ防止キャンセラー」は撤廃し、ベクトルの矛盾検知のみによるシンプルな補正（キャンセラー論理修正版）でFIXし、 $\alpha - \beta$ フィルタのパラメータ（位置ゲイン $\alpha$, 速度ゲイン $\beta$）の調整によってバランスを取る設計としました。

## 6. 進角マップの多段化と高回転域への追従

ボアアップ等によるチューニングエンジンにおいて、1速での急激な吹け上がり時に生じる高回転域（7000rpm以上）の相対的な遅角（ボコつき）を解消するため、マップ構造を拡張しました。

- **5ポイント・マップの実装**
  従来の上限であった7000rpmに加え、新たに9000rpmの進角ポイントを追加（1000, 3000, 5000, 7000, 9000rpmの5段階）。
- **サブディグリー補間の適用拡張**
  全域にわたる「1/16度（0.0625度）」の固定小数点・線形補間ロジックを、7000〜9000rpmの新規領域にもそのまま拡張適用。ハードウェア限界の滑らかさを保ったまま、トップエンドまでスムーズな進角特性と燃焼効率の確保を目指しました。
