# ほいほい堂ベース Arduino Nano AC-CDI 開発技術リファレンス
本ドキュメントは、8ビットマイコン（ATmega328P / 16MHz）という限られたリソースの中で、点火ジッターの根絶と滑らかな進角制御を実現するために投入された技術的詳細を記録したものです。

## 1. 割り込み処理（ISR）とマイコン負荷の極限最適化
- **ネイティブ・アセンブラ記述による `ISR_NAKED`**
  点火パルスの検知から出力までのレイテンシを極限まで削るため、コンパイラによる自動レジスタ退避を無効化し、アセンブラで必要最小限のレジスタ（`r16`, `r17`, `r18`）のみをスタックに退避（`push`/`pop`）する構造を採用。
- **多重割り込みの完全排除**
  高回転時にパルスが踊る（ジッターが発生する）原因が「割り込み処理中の別割り込み発生（スタックオーバーフロー）」であることを特定。全ISRの末尾の `sei` 命令を削除し、安全性を確保。

## 2. 8ビットマイコンの限界突破：固定小数点演算ロジック
浮動小数点演算（FPU）を持たないNanoで `float` や `long` の除算を行うと、数百サイクルの処理落ちが発生します。これを防ぐため、すべての演算を「乗算とビットシフト」に置き換えました。

- **直線補間の固定小数点化（角度計算）**
  従来：`(tmap[1] - tmap[0]) * (rpm - 1000) / 2000`
  最適化：除算を排除し、定数を乗じてシフトする方式へ変更。さらに Ver6.8 にて1/16度（0.0625度）の解像度（サブディグリー）を持たせるため `>> 16` から `>> 12` へ拡張し、整数切り捨てによる階段状の量子化ノイズ（飛び飛びのパルス）を根絶。全回転域においてシームレスな進角を実現。
- **ディレイ値（タイマー幅）の固定小数点化**
  従来：`sdeg * averagetime / 360`
  最適化：182 / 65536 ≒ 1 / 360 であることを利用し、16ビット固定小数点定数「182」を採用。
  数式：`((long)sdeg * averagetime * 182) >> 16` （Ver6.8のサブディグリー化に伴い `>> 20` へ調整）。

## 3. 高解像度α-βフィルタ（仮想フライホイールモデル）
単気筒エンジン特有の「燃焼行程（速い）」「排気行程（遅い）」の脈動やノイズを吸収するため、定常状態推定フィルタ（α-βフィルタ）を実装しました。

- **パラメータ設定**：位置ゲイン α = 1/4、速度ゲイン β = 1/16。
- **高解像度化（桁落ち防止）**：ArduinoのTimer1分解能（4μs/count）では、微小な加速が `>> 4` 等のビットシフトによってゼロに切り捨てられ（不感帯）、結果として「1秒ごとのパルス飛び」が発生していました。これを防ぐため、内部変数を256倍（`<< 8`）スケールで保持・計算し、出力時に `>> 8` で戻すことで、浮動小数点を使わずに完全な連続性を確保しました。

## 4. 進角タイマー（`widthB`）の位相遅れと「2手先予測」
CDIの進角は「現在のパルスを起点とし、ほぼ1回転待ってから発火する」仕組みです。そのため、計算に必要なのは「次のパルスまでの時間」ではなく、「次の次のパルスまでの時間」となります。

- **位相遅れの解消**：1手先の予測では、減速時にタイマーが短くなりすぎ、境界領域で30〜40μsの「とび（ギャップ）」や「左側への張り付き（過進角）」を引き起こしていました。
- **解決策**：フィルタの速度成分（`velocity`）を2倍して足し込む「2手先予測」を実装。
  数式：`predicted_interval = (ab_interval_fp + 2 * ab_velocity_fp) >> 8;`
  これにより、1サイクル分の遅れが数学的に完全に相殺されました。

## 5. ハードウェア慣性とシミュレーションの境界（トレードオフの理解）
ベンチテストにおいて、2500rpm → 4500rpm（3秒間）などの急激なスイープでスロットルを全閉（折り返し）にした際、フィルタの「猛烈な加速の慣性」が暴走して一時的に左へ大ジャンプする現象が確認されました。

- **ソフトウェアの限界と物理的現実**：
  ジャンプを防ぐための強烈なキャンセラーを実装すると、今度はα-βフィルタの学習が阻害され「左への張り付き」が再発するトレードオフ（デグレード）に直面しました。
  しかし、シグナルジェネレータ（慣性ゼロ）とは異なり、実際のエンジンには「重いフライホイール」が存在します。物理法則上、4500rpmから一瞬でV字に折り返して減速することはあり得ず、U字の滑らかな減速曲線を描くため、実車環境においてはこの「変曲点ジャンプ」は発生しないか、極めて小さくなります。
- **結論**：ソフトウェア上での過度な「ジャンプ防止キャンセラー（過学習）」は撤廃し、ベクトルの矛盾検知のみによるシンプルな補正（Ver 6.8 キャンセラー論理修正版）でFIXとし、実車のハードウェア慣性とのバランスを取る設計としました。

## 6. 進角マップの多段化と高回転域への追従（Ver6.9拡張）
98ccボアアップ＋PE24キャブなどの高出力チューンエンジンにおいて、1速での急激な吹け上がり時に生じる高回転域（7000rpm以上）の相対的な遅角（ボコつき）を解消するため、マップ構造を拡張しました。

- **5ポイント・マップの実装**: 従来の上限であった7000rpmに加え、新たに9000rpmの進角ポイントを追加（1000, 3000, 5000, 7000, 9000rpmの5段階）。
- **サブディグリー補間の適用拡張**: Ver6.8で確立した全域にわたる「1/16度（0.0625度）」の固定小数点・線形補間ロジックを、7000〜9000rpmの新規領域にもそのまま拡張適用。ハードウェア限界の滑らかさを保ったまま、トップエンドまでスムーズな進角特性と燃焼効率を確保しました。
